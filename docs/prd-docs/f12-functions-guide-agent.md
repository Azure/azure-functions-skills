# F12: functions-guide Agent

**Status:** 📋 Proposed  
**Draft Spec Section:** 5.2  
**Depends on:** F1 (Skill Graph Metadata)

## Problem

Individual skills handle specific tasks, but no component understands the user's **overall journey**. A developer might ask a vague question like "I want to build a serverless API" — this doesn't map to a single skill. The system needs an intelligent router that interprets user intent, considers their current context, and routes to the right skill.

## Feature

`functions-guide` is a **custom agent** (not a skill) that acts as the orchestration layer for the skill graph. It:

1. Interprets user intent from natural language
2. Evaluates the user's current context (environment, project state)
3. Selects the best skill or sequence of skills
4. Provides transitional guidance between skills
5. Handles ambiguous requests by asking clarifying questions

## Why an Agent, Not a Skill

| Aspect | Skill | Agent |
|--------|-------|-------|
| Scope | Single task | Multi-task routing |
| Context | Own execution only | All skills' metadata |
| Persistence | Stateless | Can maintain conversation context |
| Routing | Fixed transitions | Dynamic interpretation |
| Invocation | Explicit by name | Implicit via natural language |

## Routing Table

The agent uses the skill graph manifest (generated by build, F14) to route:

| User Intent | Detected Context | Routed To |
|-------------|-----------------|-----------|
| "Help me get started" | No tools installed | azure-functions-setup |
| "Help me get started" | Tools ready, no project | azure-functions-create |
| "Help me get started" | Project exists | azure-functions-discovery |
| "Deploy my app" | Project exists | azure-functions-deploy |
| "Deploy my app" | No project | azure-functions-create → azure-functions-deploy |
| "My function is slow" | Deployed project | azure-functions-observability |
| "What hosting plan?" | Any | azure-functions-hosting |
| "Python help" | Python project | azure-functions-python |
| "Build a Durable workflow" | Any | azure-functions-durable |
| "What can I do next?" | After azure-functions-setup | azure-functions-create (from graph) |

## Agent Definition

### GHCP Custom Agent

```yaml
name: functions-guide
description: Azure Functions development guide — routes you to the right skill based on your goals and project context
instructions: |
  You are the Azure Functions development guide. Your job is to understand
  what the user wants to accomplish and route them to the right skill.

  Available skills: azure-functions-help, azure-functions-setup, azure-functions-discovery, azure-functions-create, azure-functions-deploy,
  azure-functions-observability, azure-functions-hosting, azure-functions-python, azure-functions-node, azure-functions-dotnet, azure-functions-durable,
  azure-functions-feedback.

  Routing rules:
  1. New user / unclear intent → azure-functions-help
  2. Environment issues → azure-functions-setup
  3. Existing project analysis → azure-functions-discovery
  4. New project → azure-functions-create
  5. Deployment → azure-functions-deploy
  6. Monitoring / debugging → azure-functions-observability
  7. SKU / pricing questions → azure-functions-hosting
  8. Language-specific → azure-functions-python / azure-functions-node / azure-functions-dotnet
  9. Orchestration → azure-functions-durable
  10. After task completion → recommend next skill from graph

  Always explain WHY you're routing to a skill.
  If unsure, ask one clarifying question (max 1).
```

## Conversation Flow Example

```
User: "I want to build a serverless API with Python"

Agent: I can help you build a Python serverless API with Azure Functions.
       Let me check your environment first.

       [Invokes azure-functions-setup — checks pass]

       Your environment is ready (Azure CLI ✅, Core Tools ✅, Python 3.12 ✅).

       Next, let's create your project:
       → Invoking azure-functions-create with Python + HTTP trigger template

       [azure-functions-create completes]

       Your API project is scaffolded. Here are your next options:
       1. azure-functions-deploy — Deploy to Azure (recommended)
       2. azure-functions-observability — Set up monitoring before deployment
       3. azure-functions-python — Learn Python-specific patterns for your API
```

## Build Integration

The build system (F14) generates the agent's routing table from skill graph metadata:

1. Read all `graph.yaml` files
2. Build intent → skill mapping
3. Build context → skill mapping
4. Embed as agent instructions
5. Output per-target agent definitions
