# F13: post-task-suggester Hook

**Status:** 📋 Proposed  
**仮スペック Section:** 5.3  
**Depends on:** F1 (Skill Graph Metadata), F14 (Build System)

## Problem

After completing a task, developers often stop — they don't know what to do next. The skill graph has transition data, but it's only used when a user actively asks for help. There's no **push-based** mechanism that surfaces the next logical step at the moment of completion.

## Feature

`post-task-suggester` is a **hook** (not a skill) that fires after each skill completes and surfaces up to 3 next-step candidates from the skill graph. It leverages the hooks system available in GitHub Copilot CLI and Claude Code.

## Hook Triggers

| Trigger Point | Skill Completed | Suggestions |
|--------------|----------------|-------------|
| After setup | af-setup | "Next: `af-create` — Create a Functions app" |
| After create | af-create | "Next: `af-deploy` — Deploy to Azure" |
| After deploy | af-deploy | "Next: `af-observability` — Set up monitoring" |
| After discovery | af-discovery | Context-dependent top 3 from graph |
| After observability | af-observability | "Next: `af-feedback` — Share your experience" |
| After hosting | af-hosting | "Next: `af-deploy` — Deploy with selected plan" |

## Output Format

```
───────────────────────────────────────
✅ af-setup completed successfully

  Suggested next steps:
  1. af-create — Your environment is ready. Create a new Azure Functions app.
  2. af-help — Browse all available skills and guidance.

  Run a skill name to continue, or ignore to proceed manually.
───────────────────────────────────────
```

## Hook Implementation

### GitHub Copilot CLI

GHCP supports hooks that run shell commands at specific points:

```yaml
# .github/hooks/post-skill.yml
hooks:
  post_skill_completion:
    command: "node .github/scripts/suggest-next.js"
    args:
      - "--completed-skill"
      - "${SKILL_ID}"
    output: "suggestion"
```

### Claude Code

Claude Code supports hooks in its configuration:

```json
{
  "hooks": {
    "post_task": {
      "script": "suggest-next.js",
      "args": ["${completed_skill_id}"]
    }
  }
}
```

### Codex

Codex agents support post-task instructions:

```yaml
post_task_instructions: |
  After completing any skill, check the skill graph for next steps.
  Present the top 3 suggestions with reasons.
```

## Suggestion Logic

The hook reads the graph manifest (generated by build, F14) and:

1. Looks up the completed skill's `suggestions.on_success` (or `on_failure`)
2. Sorts by `priority` descending
3. Takes top 3
4. Formats with skill title + reason

```javascript
// suggest-next.js (simplified)
import { readFileSync } from 'fs';

const graphManifest = JSON.parse(readFileSync('.functions-skills/graph-manifest.json'));
const completedSkill = process.argv[2];
const outcome = process.argv[3] || 'success';

const skill = graphManifest.skills[completedSkill];
const suggestions = skill.suggestions[`on_${outcome}`]
  .sort((a, b) => b.priority - a.priority)
  .slice(0, 3);

console.log(`\n✅ ${completedSkill} completed successfully\n`);
console.log('  Suggested next steps:');
suggestions.forEach((s, i) => {
  console.log(`  ${i + 1}. ${s.target} — ${s.reason}`);
});
```

## UX Rules

1. **Maximum 3 suggestions** — more creates decision fatigue
2. **Always show reason** — "Deploy to Azure" alone is not enough; explain why it's the logical next step
3. **Non-blocking** — suggestions are informational; the user can ignore them
4. **No repeated suggestions** — if a skill was already completed in this session, don't suggest it again
5. **Failure-aware** — if the previous skill failed, show `on_failure` suggestions instead

## Build Integration

The build system (F14) generates:

1. `graph-manifest.json` — complete graph for hook consumption
2. Target-specific hook scripts (GHCP, Claude, Codex formats)
3. Hook registration files per target
