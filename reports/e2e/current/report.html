Issues Found

  • CRITICAL Old version (0.0.2) creates massive instruction files (52–53KB)
    Affected: TC-S2-GHCP-LOCAL, TC-S2-CLAUDE-LOCAL, TC-S2-CODEX-LOCAL
    The old version (0.0.2-preview) merges all skill content into copilot-instructions.md / CLAUDE.md / AGENTS.md, creating 52–53KB files. The current update command does NOT slim these files — it creates save-aside files instead. These massive files are always loaded into every LLM request, severely impacting performance and token costs.
  • HIGH GHCP plugin mode: 0 skills loaded in session
    Affected: TC-S1-GHCP-PLUGIN (S1GP-10)
    In plugin mode, session.skills_loaded events show 0 azure-functions-* skills. Only customize-cloud-agent appears. This is likely a GitHub Copilot limitation where skills bundled in plugins are loaded differently, but it means plugin-mode users may not see skills in the session UI.
  • HIGH GHCP plugin agent name resolution failure
    Affected: TC-S1-GHCP-PLUGIN (S1GP-11)
    copilot --agent azure-functions-skills:functions-copilot returns "No such agent". Direct invocation of the plugin's agent definition does not work. Users must use the agent through the normal chat flow, not direct invocation.
  • MEDIUM Codex sandbox errors during chat inspection
    Affected: TC-S1-CODEX-LOCAL (S1XL-9), TC-S1-CODEX-PLUGIN (S1XP-8)
    Codex chat exits 0 but internal shell probes fail with "ERROR windows sandbox: spawn setup refresh". The LLM still produces a metadata-based summary, but filesystem inspection commands fail in the sandbox. This is a Codex runtime limitation on Windows, not a CLI bug.
  • MEDIUM Old version codex plugin install fails with access denied
    Affected: TC-S2-CODEX-PLUGIN (S2XP-2)
    npx @azure/functions-skills@0.0.2-preview install --agent codex fails with "failed to back up plugin cache entry: Access is denied (os error 5)". This is a Codex marketplace cache permission issue with the old version. The current version (0.0.3) does not have this issue.
  • LOW DEP0190 warning in old version
    Affected: TC-S2-GHCP-PLUGIN (S2GP-2)
    Old version (0.0.2) emits DEP0190 DeprecationWarning on Node.js v24. This was fixed in the current version. No action needed — the warning only appears when installing from the old published package.
  • INFO Codex hooks use bash on Windows
    Affected: TC-S1-CODEX-LOCAL (S1XL-6)
    The .codex/hooks.json startup hook command uses bash -c '...'. On Windows systems without WSL/Git Bash, this hook may fail silently. Consider using cross-platform commands or detecting the OS.
  • INFO Update creates save-aside files for user-modifiable content
    Affected: TC-S2-GHCP-LOCAL, TC-S2-CLAUDE-LOCAL, TC-S2-CODEX-LOCAL
    When files like AGENTS.md, mcp.json, or config.toml differ between old and new versions, update creates *.azure-functions-skills-new.* files. This is by design (non-destructive update) but means users must manually merge changes.
