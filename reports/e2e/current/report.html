Failure Analysis

GitHub Copilot plugin agent missing

Fresh lifecycle evidence is complete: copilot plugin uninstall azure-functions-skills, marketplace removal, marketplace add, plugin install, and post-list all succeeded. The install reported Installed 9 skills. The required inspection command failed with No such agent: azure-functions-skills:functions-copilot, available:. This suggests the default plugin payload is skills-only or the manifest does not expose a qualified agent surface to Copilot CLI.

Recommended fix: either include the functions-copilot agent in the plugin payload expected by this scenario, or update the scenario to validate skills-only plugin payloads separately from workspace/full-profile agent activation.

Chat scenario mismatch

Claude and Codex chat launchers delivered Azure Functions startup prompt context and detected host.json, but neither wrote workspace-local skills, MCP, hooks, or agent guidance files. GitHub Copilot chat failed earlier because functions-copilot was not available through the chat launcher. The current README says chat is prompt-only, while the scenario catalog requires auto-setup.

Recommended fix: align references/scenarios.md with current prompt-only chat semantics, or restore chat auto-setup before the agent launch.

Codex setup warning

Codex inspected the setup workspace and listed all 9 skill names, but emitted skills.count: 8. One recursive PowerShell listing was blocked by policy, then Codex recovered with an alternate listing. The scenario is marked warning rather than pass because the durable artifact is internally inconsistent.
