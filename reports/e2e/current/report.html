Key Findings

1. GitHub Copilot chat and plugin inspection use different agent ids

The original command evidence mixed the setup target id and plugin agent id into the chat scenario. The corrected chat command uses azure-functions-skills chat --agent github-copilot plus headless Copilot passthrough, which lets Copilot load the workspace-local functions-copilot agent. Installed-plugin inspection is separate and correctly uses copilot --agent azure-functions-skills:functions-copilot.

2. Claude/Codex agent guidance is target-specific

Claude should not be marked false for missing agents metadata: the build code emits CLAUDE.md for workspace setup, and the Claude plugin manifest intentionally omits unsupported agents and interface fields. Codex workspace setup similarly carries guidance through AGENTS.md, while the Codex plugin manifest declares agents but did not expose functions-copilot as an active custom-agent surface in this session.

3. Codex install is stricter than the README shows

codex plugin add azure-functions-skills failed because Codex requires a marketplace qualifier. codex plugin add azure-functions-skills@azure-functions-skills succeeded, installed version 0.11.3, and exposed all 9 skills plus Azure MCP.

4. Help probes can mutate the repository root

Early discovery commands for setup --help and chat --help created root setup/chat artifacts. The run removed only untracked generated artifacts and kept tracked user changes intact.
