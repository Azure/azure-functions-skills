Failure Analysis

Windows chat launcher failures

src/chat/index.ts resolves launchers with where. On this machine, direct PowerShell discovery finds copilot.ps1 and codex.ps1, but chat fails with copilot not found and codex not found. The resolver only treats .cmd and .bat as shell-backed commands, so PowerShell shims are not launched correctly.

Claude chat blocks on MCP selection

The chat command installs Claude files and launches Claude, but Claude displays an interactive MCP server selection prompt for azure-functions-templates and azure. That blocks unattended E2E completion.

Plugin lifecycle cannot be proven

Fresh plugin install semantics require pre-state, cleanup/isolation, install/register, post-state, and agent inspection. None of the plugin scenarios reached that full lifecycle noninteractively, so none are marked pass.

Docs mismatch

README skill and setup/chat examples mostly match the repository, but the Claude plugin-from-source command is not supported by the current CLI help observed in this run. README also does not warn about Windows PowerShell shim launch behavior for chat.
